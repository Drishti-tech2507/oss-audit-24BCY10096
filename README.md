## Project Overview

This project is an open source audit of LibreOffice, examining its origin, license, ethics, Linux footprint, and ecosystem. The project also includes five shell scripts that demonstrate Linux system administration and automation skills.

---

## Scripts Included

| Script | Purpose |
|--------|---------|
| `system_info.sh` | Displays system identity including OS, kernel, user, uptime |
| `package_check.sh` | Checks if LibreOffice is installed and displays philosophy note |
| `disk_audit.sh` | Audits system directories for size and permissions |
| `log_analyzer.sh` | Analyzes log files for error keywords |
| `manifesto.sh` | Generates a personalized open source philosophy statement |

---

## How to Run the Scripts

### Step 1: Make Scripts Executable
```bash
chmod +x *.sh
```

### Step 2: Run Each Script

**Script 1 - System Identity Report**
```bash
./system_info.sh
```

**Script 2 - FOSS Package Inspector**
```bash
./package_check.sh
```

**Script 3 - Disk and Permission Auditor**
```bash
./disk_audit.sh
```

**Script 4 - Log File Analyzer**
```bash
sudo ./log_analyzer.sh /var/log/dpkg.log.1 error
```

**Script 5 - Open Source Manifesto Generator**
```bash
./manifesto.sh
```

## Dependencies

- **Operating System:** Linux (Kali Linux, Ubuntu, Debian, Fedora, or any Linux distribution)
- **Shell:** Bash (version 4.0 or higher)
- **Required Packages:** 
  - LibreOffice (for Script 2)
  - sudo access (for Script 4 to read system logs)
- **Package Managers:** apt (Debian/Ubuntu/Kali) or rpm (Fedora/RHEL)

---

## Installation Instructions

### For Kali Linux / Ubuntu / Debian:

```bash
# Update package list
sudo apt update

# Install LibreOffice (if not already installed)
sudo apt install libreoffice -y

# Make scripts executable
chmod +x *.sh
```

### For Fedora / RHEL:

```bash
# Install LibreOffice
sudo dnf install libreoffice

# Make scripts executable
chmod +x *.sh
```

## Project Report Summary

The full report (12-16 pages) covers the following sections:

### Part A: Origin and Philosophy
- **A1:** Origin story of LibreOffice and the OpenOffice.org fork
- **A2:** License analysis (MPL 2.0) and the four freedoms
- **A3:** Ethical reflection on corporate control vs community governance

### Part B: Linux Footprint
- Installation methods and commands
- Key directories (binaries, config files, logs)
- User and group permissions and security implications
- Update model and package management

### Part C: FOSS Ecosystem
- Dependencies (GTK, Python, HarfBuzz, ICU)
- Forks and spin-offs (Collabora Office)
- Community governance (The Document Foundation)

### Part D: Open Source vs Proprietary
- Comparison with Microsoft Office
- Analysis of cost, security, support, and freedom
- Recommendation for real-world deployment
